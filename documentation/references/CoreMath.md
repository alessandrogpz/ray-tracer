# Reference: Core Mathematics

This section provides technical details regarding the foundational mathematical structures used in the ray tracer. These structures are essential for representing points, vectors, colors, and transformations in 3D space.

## Tuples

A `Tuple` is the fundamental data structure used to represent coordinates and directions in 3D space. It is composed of four floating-point numbers: `x`, `y`, `z`, and `w`.

- **`x, y, z`**: Represent the position or magnitude along the three axes.
- **`w`**: The homogeneous coordinate. It is crucial for distinguishing between points and vectors and for allowing matrix translations.

### Points vs. Vectors

- **Point (`w = 1.0`)**: Represents a specific location in space. When `w=1`, the point can be translated (moved) by matrix multiplication.
- **Vector (`w = 0.0`)**: Represents a direction and magnitude, but no specific location. When `w=0`, translation matrices have no effect on the vector, which is mathematically correct (a direction doesn't change when you move it).

*Reference code: `src/Tuple.cppm`*

## Colors

Colors in the ray tracer are implemented similarly to Tuples, but they represent the Red, Green, and Blue (RGB) components of light.

- **`red, green, blue`**: Floating-point values typically ranging from `0.0` (no intensity) to `1.0` (full intensity).
- Operations on colors include addition (combining light), subtraction, and scalar multiplication (adjusting brightness).

*Reference code: `src/Colors.cppm`*

## Matrices

A `Matrix` is a grid of numbers used to manipulate Tuples. In 3D graphics, a 4x4 matrix is standard because it allows for homogeneous coordinates (the `w` component in our Tuples).

### Transformations

Transformations are specialized matrices that alter a Tuple when multiplied together (`Matrix * Tuple`). 

The core transformations are:
- **Translation**: Moves a point in space. Does not affect vectors.
- **Scaling**: Changes the size of an object. Affects both points and vectors.
- **Rotation**: Rotates a point or vector around the X, Y, or Z axis.
- **Shearing**: Slants the object by shifting points along one axis based on their coordinate on another axis.

Transformations can be chained together via matrix multiplication. The order of multiplication is crucial: `Transformation_3 * Transformation_2 * Transformation_1 * Point` applies Transformation 1 first, then 2, then 3.

*Reference code: `src/Matrix.cppm`, `src/Transformations.cppm`*

## Canvas

A `Canvas` is a rectangular grid of pixels, where each pixel is a `Color`. It represents the final image generated by the ray tracer. Once rendering is complete, the `Canvas` is typically exported to a standard image format like PPM (Portable Pixmap).

*Reference code: `src/Canvas.cppm`*
